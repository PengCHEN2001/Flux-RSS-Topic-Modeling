08/04/2026 — Constitution et test du corpus

Objectif

Générer un corpus sérialisé afin de tester le script run_bertopic.py de mon collègue.

1. Constitution du corpus brut

Le corpus brut a été constitué à partir des fichiers XML RSS, à l'aide du script rss_parcours.py avec la méthode feedparser. Le corpus obtenu contient 3210 articles issus de plusieurs sources (Libération, etc.), sérialisés au format JSON.

2. Enrichissement morphosyntaxique

Le corpus a été enrichi morphosyntaxiquement grâce au script analyzers.py en utilisant spaCy (fr_core_news_md). Chaque article a été tokenisé et annoté en lemmes et en parties du discours (POS).

3. Modélisation thématique par LDA

Une modélisation par LDA a été lancée via run_lda.py sur les lemmes filtrés par catégories grammaticales (NOUN, VERB, ADJ), avec extraction de 10 topics et activation des bigrammes. Résultats sauvegardés dans topics.json.

4. Test de run_bertopic.py

Le script de mon collègue a été testé avec la commande suivante :

python3 run_bertopic.py corpus_analyse.json -f json -o test.html --chart 2d

BERTopic a traité les 3210 documents et extrait 10 topics. Le modèle a téléchargé automatiquement paraphrase-multilingual-MiniLM-L12-v2 depuis Hugging Face. La visualisation 2D a été générée avec succès.

Topics détectés :

  • Politique française — Macron, budget
  • Actualité internationale — Trump, droits de douane, Gaza
  • Faits divers — meurtre de Louise en Essonne
  • Sport -- Marseille
  • Société -- retraites, culture

Test avec --chart barchart :

python3 run_bertopic.py corpus_analyse.json -f json -o test_bar.html --chart barchart

Visualisation générée avec succès. Les topics varient légèrement d'une exécution à l'autre — normal, BERTopic comportant une part d'aléatoire à chaque entraînement.

git merge MBHS-ex2-4s10

Améliorations apportées

  • 4a -- --token : choisir entre les lemmes (lemma) et les mot-formes (form)
  • 4b -- --pos : filtrer sur les catégories grammaticales (ex : NOUN, VERB)
  • 4c : les deux options disponibles en ligne de commande

Exemple d'utilisation :

python run_bertopic.py -f json corpus_analyse.json --token lemma --pos NOUN VERB -o topics.html

Script testé sur corpus_analyse.json (3207 documents) — fonctionne correctement.

Exercice 1 -- Correction du code précédent

J'ai repris et restructuré une grande partie du code, notamment dans analyzers.py, datastructures.py et run_lda.py. En m'inspirant du style de la correction fournie par le professeur, j'ai supprimé du code et des classes superflus afin de rendre l'architecture plus claire et lisible.

Dans la version précédente, dans la classe Article, le champ token était utilisé comme clé avec un type limité à list[Token]. J'ai modifié cette structure : 

analysis: list[list[Token]] = field(default_factory=list)

J'ai également corrigé un bug dans load_xml de datastructures.py : l'ancienne version écrasait certaines variables (conflit de noms) et ciblait incorrectement les balises XML, produisant des listes vides. La désérialisation remplit désormais correctement la structure list[list[Token]].

Exercice 2 -- Topic Modeling avec BERTopic

Exo 2.1 -- Quick start

Exécution réussie du script pour le quick start.

Exo 2.2 -- Adaptation sur données françaises

J'ai adapté le script pour charger nos propres données en français. Problème rencontré : lors de la génération de la visualisation 2D (visualize_topics()), le programme a planté (erreur UMAP) car le corpus de test était trop petit. J'ai basculé sur visualize_barchart(), qui fonctionne parfaitement même avec peu de topics.

Observation clé : les topics étaient saturés de mots vides (« le », « de », « et »), ce qui confirme la nécessité d'un filtrage syntaxique pour la suite (exo 2.4).

Exo 2.3 -- Commande autonome

J'ai refactorisé le script pour éliminer les valeurs codées en dur, en structurant le code par fonctionnalités :

  • load_corpus() — charge le corpus en xml, json ou pkl
  • train_bertopic_model() — entraîne le modèle
  • save_viz() — sauvegarde la visualisation, avec bascule automatique vers barchart si la vue 2D échoue

Options ajoutées avec argparse : -f pour le format du corpus, -o pour sauvegarder le graphique HTML. Exemple : 

python run_bertopic.py -f xml Outputs/corpusA.xml
